# Business Idea 写作规范 v1

## 目标

这不是公众号，不是营销稿，也不是咨询公司黑话文档。

它是一种公开研究写法，同时服务两个目标：
1. 帮项目维护者每天获得一个值得判断的创业 idea / 公司 / 方向
2. 让外部读者也能快速理解这个项目为什么值得关注

## 文风原则

### 1. 先真实，再流畅
- 不确定就说不确定
- 没查到就说暂缺
- 不用猜测冒充事实
- 不用行业黑话掩盖信息不足

### 2. 写判断，不写空话
不要只写“这家公司很有潜力”“赛道很大”。
要写清楚：
- 为什么值得看
- 具体强在哪
- 风险在哪
- 为什么现在看它，而不是别家

### 3. 像认真研究的人，不像营销号
允许有观点，但不要表演人格。
允许有判断，但不要装权威。
允许有兴奋感，但不要写成喊单。

### 4. 统一结构，方便长期积累
每篇公司页尽量遵循同一骨架，避免今天像日报，明天像百科，后天像投研 memo。

## 推荐页面结构

### 开头摘要
用 2-4 句话回答：
- 这是什么公司 / 项目
- 它在解决什么问题
- 为什么这次值得看

### 这家公司在做什么
写清楚：
- 产品是什么
- 面向谁
- 核心使用场景是什么
- 和常见印象中的同类产品有什么不同

### 为什么值得关注
优先写 3 条左右，必须具体。
可以从这些角度写：
- 产品切口
- 用户价值
- 市场时机
- 增长信号
- 团队背景
- 商业化潜力

### 投资人视角批判
至少写 2 条。
不要假装所有公司都很好。
重点看：
- 增长是否可能只是短期热度
- 产品护城河是否真的存在
- 市场是否拥挤
- 商业模式是否成立
- 团队执行难度是否被低估

### 我们的判断
用一小段写清楚：
- 现在这家公司最值得盯什么
- 它更像短期热点，还是长期可跟踪标的
- 现在该保持什么态度

### 可信来源
优先列出：
- 官网
- 官方博客 / 官方文档
- Wikipedia
- 主流可信媒体
- Hacker News（可作为线索源，不单独当最终事实源）

## 语言要求

### 应该多用
- 具体公司名、产品名、创始人名
- 具体场景
- 具体信号
- 具体担忧
- 具体比较对象

### 应该少用
- 赋能、闭环、抓手、生态位卡位这种空词
- “意味着什么”“本质上”“说白了”这种 AI 常见句式
- “首先其次最后”这种课堂结构词

## 句子风格
- 句子尽量短一些
- 可以有观点，但别装腔
- 一段只讲一个重点
- 让读者能快速扫读

## 文章来源要求

### 可接受的一手/强来源
- 公司官网
- 官方产品页
- 官方博客
- 官方 X / LinkedIn / GitHub
- 权威数据库 / 权威媒体原创报道

### 可接受的线索源
- Hacker News
- Product Hunt
- Reddit / X 讨论

### 不推荐直接当主来源
- 营销软文
- 无出处转载
- 标题党媒体
- 纯二手整理站

## 重写目标
重写旧的 daily-pick 时，不保留“日报腔”。
要改成可以长期留在站里的公司页。

## 最终标准
一篇合格的公司页应该让人读完后能回答：
- 这公司是谁
- 它在做什么
- 为什么现在值得看
- 为什么又不能盲目乐观
- 如果以后继续跟踪，应该盯什么
