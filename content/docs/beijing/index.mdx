# 北京

北京总览承担城市入口的作用：先回答北京有哪些典型互联网 / 科技公司、它们大致分布在哪些办公区、当前已经收录了哪些公司指南，再进入具体租房页面。

## 怎么使用这个总览

1. 先在地图里判断公司属于哪条通勤带：海淀北部、北三环、望京/来广营、朝阳中部、亦庄，还是更外侧的昌平 / 顺义。
2. 再看生态信息表里的“收录”状态，已收录公司可以直接进入对应租房指南；后续如扩展新样本，可先按同片区页面判断。
3. 如果目标公司不在当前样本里，先按同一片区的已收录公司理解通勤逻辑，再核验自己的团队办公点。

## 地图判断原则

北京租房不能只看直线距离。尤其是海淀北部和后厂村一带，最后 1 到 2 公里的骑行、摆渡车、打车可得性，经常比“离地铁几站”更影响体感。

- **海淀北部**：百度、小米、快手、滴滴、腾讯、金山办公、用友、地平线等更依赖上地、西二旗、西北旺、清河、回龙观一线。
- **北三环 / 学院路 / 中关村**：字节跳动、知乎、搜狐、汽车之家、贝壳等更适合从大钟寺、知春路、五道口、学院路判断。
- **望京 / 来广营 / 酒仙桥**：阿里、美团、陌陌、360、完美世界等要区分望京办公点、崔各庄 / 来广营新园区、北苑和酒仙桥办公带。
- **朝阳中部 / 东城**：微博、爱奇艺、BOSS 直聘、昆仑万维等更看重地铁换乘和晚归便利。
- **亦庄**：京东总部更像独立通勤题，要把亦庄线、班车和周边生活配套一起算。
- **昌平 / 顺义**：好未来、理想汽车这类更外侧办公点要先确认班车、自驾、地铁末端接驳，再谈租房半径。

## 北京找房方式

<Callout type="info" title="北京的核心判断">
北京租房要先看 **通勤带**，再看 **距离圈层**，最后才看单套房源。北三环、后厂村、望京、亦庄的通勤逻辑完全不同，不能用同一套距离标准判断。
</Callout>

### 到京前怎么安排

如果还没在北京落脚，建议提前到京住酒店，再集中看房。酒店优先选在目标办公区附近，或者选在对应地铁线附近，方便白天看采光、晚上复看噪音和回家路线。

- **北三环 / 中关村**：优先住大钟寺、知春路、牡丹园、五道口附近
- **海淀北部 / 后厂村**：优先住上地、西二旗、清河、回龙观附近
- **望京 / 来广营**：优先住望京、望京东、来广营、崔各庄附近
- **亦庄**：优先住荣京东街、万源街、经海路、次渠附近

## 资料来源与口径

公司定位优先使用官网、投资者关系页、年报、政府公开信息；租金和小区建议放在各公司指南中单独维护。当前北京样本已全部形成独立租房指南，后续如果扩展新公司，会继续按同一来源标准校准办公点和通勤圈层。

## 友情推荐

这里用于沉淀北京城市级的靠谱中介、公司内同事转租、合租室友和公寓管家信息。当前先按推荐类型管理，不直接展示未经核验的私人联系方式。

| 推荐类型 | 适合场景 | 推荐标准 | 使用建议 |
| --- | --- | --- | --- |
| 公司内同事 / 校友转租 | 短租、合租、快速过渡 | 能提供原合同、剩余租期、费用明细 | 优先找同团队、校友群、可信转介绍 |
| 熟悉目标片区的门店中介 | 想集中看多个小区 | 熟悉办公区周边 3 到 5 公里通勤圈 | 只看能提供真实房号和合同主体的房源 |
| 品牌公寓管家 | 想省心、可接受溢价 | 费用透明、退租规则清楚、维修响应稳定 | 问清服务费、短租限制和空气质量记录 |
| 个人转租平台房源 | 预算敏感、愿意花时间筛 | 房东授权清楚，拒绝先付定金看房 | 必须线下看房，签约前核验身份和授权 |
