# 北京

北京总览承担城市入口的作用：先回答北京有哪些典型互联网 / 科技公司、它们大致分布在哪些办公区、当前已经收录了哪些公司指南，再进入具体租房页面。

## 怎么使用这个总览

1. 先在地图里判断公司属于哪条通勤带：海淀北部、北三环、望京/来广营、朝阳中部、亦庄，还是更外侧的昌平 / 顺义。
2. 再看生态信息表里的“收录”状态，已收录公司可以直接进入对应租房指南。
3. 如果公司还未开独立页面，先按同一片区的已收录公司理解通勤逻辑，再核验自己的团队办公点。

## 地图判断原则

北京租房不能只看直线距离。尤其是海淀北部和后厂村一带，最后 1 到 2 公里的骑行、摆渡车、打车可得性，经常比“离地铁几站”更影响体感。

- **海淀北部**：百度、小米、快手、滴滴、腾讯、金山办公、用友、地平线等更依赖上地、西二旗、西北旺、清河、回龙观一线。
- **北三环 / 学院路 / 中关村**：字节跳动、知乎、搜狐、汽车之家、贝壳等更适合从大钟寺、知春路、五道口、学院路判断。
- **望京 / 来广营 / 酒仙桥**：阿里、美团、陌陌、360、完美世界等要区分望京办公点、崔各庄 / 来广营新园区和酒仙桥办公带。
- **朝阳中部 / 东城**：微博、爱奇艺、BOSS 直聘、昆仑万维等更看重地铁换乘和晚归便利。
- **亦庄**：京东总部更像独立通勤题，要把亦庄线、班车和周边生活配套一起算。
- **昌平 / 顺义**：好未来、理想汽车这类更外侧办公点要先确认班车、自驾、地铁末端接驳，再谈租房半径。

## 资料口径

公司定位优先使用官网、投资者关系页、年报、政府公开信息；租金和小区建议放在各公司指南中单独维护。生态表里的“待补充”表示已纳入北京大厂地图视野，但暂未形成独立租房指南，不代表推荐或不推荐。
